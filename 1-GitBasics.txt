1. git init – Initializes a new Git repository so Git can start tracking changes.

2. git clone – Creates a local copy of an existing remote repository from GitHub or GitLab.

3. git add – Moves selected files from the working directory to the staging area before committing.

4. git commit – Saves a snapshot of staged changes in the local repository with a meaningful message.

5. git push – Uploads local commits to a remote repository (such as GitHub) so others can access the code.
